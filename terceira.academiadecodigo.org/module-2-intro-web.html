World Wide Web

System developed for information sharing across internationally dispersed teams, originally aimed at the High Energy Physics community

Stores documents and other web resources, interlinked with each other and accessed via the Internet

World Wide Web

In 1989, Tim Berners Lee invented the three fundamental technologies that are the foundation of the WWW

  • HTTP - HyperText Transfer Protocol
  • URI - Unique Resource Identifier
  • HTML - HyperText Markup Language

Not only that, he wrote the first web browser and first web server

After that, he created and still leads the World Wide Web Consortium (W3C)

Uniform Resource Locators

A pointer to a resource on the World Wide Web. Can be something as simple as a file or a directory, or a reference to a more complicated object, such as a query to a database or to a search engine

A Web HTML document can contain a number of URLs, pointing to other Web resources (links)

Hypertext Transfer Protocol

HTTP is a stateless, application-layer protocol for communicating between distributed systems and is the foundation of the modern web. Communication between a host and a client occurs via a request/response pair. The client initiates an HTTP request message, which is serviced through an HTTP response message in return.

Modern web apps use state, but they need to use other mechanisms to store state, as HTTP does not support it: (eg: cookie based sessions)

HTTP Verbs

HTTP verbs define the action to perform on a specific web resource

The most popular verbs are:

  • GET Fetch an existing resource. The URL contains all the necessary information the server needs to locate and return the resource
  • POST Create a new resource. POST requests usually carry a payload that specifies the data for the new resource
  • PUT Update an existing resource. The payload may contain the updated data for the resource
  • DELETE Delete an existing resource

HTTP Status Codes

With URLs and verbs, the client can initiate requests to the server. In return, the server responds with status codes and message payloads. The status code is important and tells the client how to interpret the server response

HTML

HyperText Markup Language

The most basic building block of the Web.

Made up of elements that may be used to:

  • Structure the web document into logical sections (navigation, header, content, footer, ...etc)

  • Convey meaning to different pieces of text (title, subtitle, paragraph, list, ..etc)
  • Extensible Markup Language - a markup language that defines a set of rules for encoding documents in a format that is both human and machine-readable.

  • Closing Tag - States where the element ends or stops taking effect

Allows the structure of a web page to be separated from its presentation and styling

Browser Defaults

You've already seen some styling being implemented:

Bring Your Own Styling

You may extend browser default styles using CSS rules.

Rules specify a set of styles to be applied to a group of elements.

A rule looks like this:

    font-size: 12px;
}
  • h1 is the selector: targets a group of elements
  • declarations describe styles to be applied
  • declarations are written inside brackets as property:value pairs
  • this rule states: I want all h1 elements to be colored in red and have a font-size of 12px
  • you may add as many rules as you want
  • its hard to remember all existent properties and their values, so usually we look them up on the internet everytime we need them. e.g if we wanted to find how to add a background image to an element, we would search on google 'css background image'

There are three core concepts in CSS that define which styles get applied to an element.

  • Cascading
  • Specificity
  • Inheritance

To truly master CSS, we need to understand these three concepts!

Cascading in CSS

Cascading is a core concept in CSS.

Stylesheets cascade. The order of the rules matter: the last one will have precedence over previous ones.

If a document is linked to multiple stylesheets, the last ones will have precedence over the previous ones. 

h1 {
    font-size: 10px;
}

h1 {
    font-size: 12px;
}
<h1> font-size will be 12px!

Specificity

Another core concept in CSS.

Defines which styles will be applied in case of conflicting rules.

Specificity states that more specific selectors will have precedence over broader ones.

  • element name
  • class
  • id
img {
    border-radius: 0px;
}

.round-corners {
    border-radius: 10px;
}

#ball {
    border-radius: 100%;
}
What will be the border-radius of <img id="ball" class="round-corners"/>?
  • the rules of specificity get complex when combining multiple selectors and the outcome may get hard to predict.. experimenting is usually part of the process.
  • if both rules have the same specificity, last one is applied (remember cascading?)

Inheritance

There are two kinds of properties:

  • Inherited properties
  • Non-inherited properties

Children elements inherit parent's inherited properties.

Element Boxes

HTML elements wrap content inside a box.

Web browsers render content and its box using a set of rules called CSS Box Model.

With CSS, it's possible to define how the content is positioned inside its own box and how all boxes are layed out in the document.

  • remember we've talked about block and inline boxes? those are boxes that wrap content -- exactly those we're talking about
  • before going into further details about how we can position content and their boxes, we must first understand what's the CSS Box Model

CSS Box Model

Specification that defines how element boxes are rendered

  • Content box: the area where your content is displayed. Can be sized using width and height properties.
  • Padding box: white space around content. Can be sized using padding related properties.
  • Border box: border box wraps content and padding. Can be controlled using border related properties.
  • Margin box: white space around content, padding and border box. can be controlled using margin related properties
  • theres a alternative box model that you can use... we'll talk about it in the hands on

Boxes, boxes, and more boxes

CSS broadly defines two types of boxes.

They mostly differ in how they handle parent's available space.

  • Block boxes
  • Inline boxes

The display property defines which kind of box surrounds the content.

  • We've already seen block-level and inline-level elements -- these are elements whose box is either a block or an inline box by default

Block Boxes

Break onto a new line, stacking on top of each other.

Fill available parent space (in the writing direction) by default.

Respect width and height properties.

If width is not defined, they are as wide as their parent.

Paddings, margins and borders push other elements away.

  • writing direction may be different in other languages. e.g arabic is right to left, chinese is top to bottom

Inline Boxes

Do not break onto a new line.

Width and height properties do not apply.

Element size is always the sum of content size, padding, border width and margin.

Horizontal paddings push other inline elements away.

Inline-block Boxes

There's a special kind of box that is something in-between.

It's like an inline box but width and height may be set.

If undefined, box size will be the same as an inline box. 

display: inline-block;

Generic Elements

HTML provides two generic elements for content wrapping.

These elements hold no semantic meaning, unlike <article>, <section> or <h1>.

  • <div> to wrap content in a block box
  • <span> to wrap content in an inline box

display: block;
display: inline;
display: inline-block;

display property superpowers

display property also provides two possible values that change how child elements are layed out.

These are extremely useful when creating complex layouts.

Although children's rendering flow is changed, the element itself is treated as a block. 

display: flex;
display: grid;
  • we're not gonna talk about css grids because they are still not widely supported and are a more complex than flexbox. we'll provide a learning resource later for those who are interested
  • there are other CSS techniques (some older, some newer) for creating page layouts but CSS Flexbox is the best compromise between ease of use and support right now

Flexible Box Layout (aka Flexbox)

Easily lays out elements in a single dimension (row or column). 

<div class="parent">
    <div class="child">one</div>
    <div class="important child">two</div>
    <div class="child">three</div>
</div>

.parent {
    outline: 1px solid blue;
    display: flex;
}

.child {
    outline: 1px dotted red;
}

Flexbox children flow

Flexbox provides extra properties to tune how children are layed out. 

.parent {
    flex-direction: column;
    justify-content: stretch; /* changes main axis flow */
    align-items: center; /* changes cross axis flow */
}

Flexbox children flow

Flex container children may define how much available space they fill in the main axis through the flex property.

The value may be any positive number and represents a relative size. 

.child {
    flex: 1;
    outline: 1px solid red;
}

.important {
    flex: 2;
}
  • there are more units and values you can set on the flex property, but we'll keep it simple. if you are interested in more details, there's a reference and a guide later.

Element positioning

Positioning allows moving an element from where it would be placed in normal document flow.

Suited for managing and fine-tuning a specific element's position.

There are 5 types of positioning:

  • Static positioning
  • Relative positioning
  • Absolute positioning
  • Fixed positioning
  • Sticky positioning
  • element positioning is not useful for creating page layouts. use flexbox instead

position property

Element positioning is static by default, but may be changed through the position property. 

position: static; /* default value */
position: relative;
position: absolute;
position: fixed;
position: sticky;

Static positioning

The element is positioned where it should be in the document flow. 

.parent {
    border: 2px black dotted;
    display: inline-block;
}

.box {
    display: inline-block;
    background: red;
    width: 100px;
    height: 100px;
}

#two {
    background: green;
}
<div class="parent">
    <div class="box" id="one">One</div>
    <div class="box" id="two">Two</div>
    <div class="box" id="three">Three</div>
    <div class="box" id="four">Four</div>
</div>

Relative positioning

Allows offseting an element position.

This may be done with top, left, right and bottom properties. 

#two {
    top: 20px;
    left: 20px;
    background: green;
    position: relative;
}
  • top,left,right,bottom values define the offset from the element's original position

Absolute Positioning

Removes an element from the document flow.

Element is positioned relatively to the nearest positioned parent. 

.parent {
    border: 2px black dotted;
    display: inline-block;
    position: relative;
}

#two {
    top: 20px;
    left: 20px;
    background: green;
    position: absolute;
}

Fixed Positioning

Removes an element from the document flow.

Element is positioned relatively to the viewport, which makes it look fixed in place. 

#two {
    top: 20px;
    left: 20px;
    background: green;
    position: fixed;
}

Sticky Positioning

Statically positioned item until scrolled to the desired viewport offset.

Once it hits the desired offset, it will act as a fixed position element.

Older browsers do not support this value. 

#two {
    top: 20px;
    left: 20px;
    background: green;
    position: sticky;
}

Size does matter

The advent of the web has brought browsers to an ever-growing number of devices with different screen sizes.

This brought up a question to developers and designers everywhere...

How can my web site look good on them all?

  • not only every device has a browser nowadays, but it's also the most popular way to interact with services (online or not)

Responsive Web Design

A set of practices that allows web pages to alter their layout and appearance to suit different screen sizes.

Initially based on complex legacy CSS layout techniques and media queries.

CSS has since evolved, providing tools to develop responsively easily.

Flexbox is one of those tools! Did you notice how responsive your Flexbox Layout was?
  • responsive web design its not a "technology" or "feature". its just a term to describe a set of best practices to create a layout that can respond to the device being used to view the content. Your website may be more or less responsive depending on how strictly you've followed these guidelines while developing.

Media Queries

Available around 2010, it's one of the most important CSS features to truly allow Responsive Web Design.

Media Queries allow developers to apply different CSS styles depending on viewing device's screen characteristics. 

@media screen and (min-width: 800px) { 
/* these styles will only be applied if it's a screen that's at least 800px wide */
    .container { 
        margin: 1em 2em; 
    }
}
  • Multiple media queries may be added
  • using media queries, we may completely change our page layout and appearance to fit different screen sizes
  • the sizes at which our layout changes are called breakpoints

Mobile First Development

Web is being increasily accessed through mobile devices.

Developers have found it's easier to develop a web page targeting small screens, and then use media queries to make it look good on bigger ones.

  • Last year, mobile traffic has surpassed desktop traffic. (51-49%)
  • making sure a web page looks good on a mobile device has never been more important
  • It's easier to initially get your point accross through small screens, and then add *not so important* elements to bigger screens than the opposite.

Relative Values

CSS allows usage of relative values in most size-related properties.

  • % - percentage of parent's same property value (in most cases). useful for relative width and height.
  • em - size relative to parent's font-size. useful for relative font sizes.
  • rem - size relative to root font-size. useful for relative font sizes.
  • 1em is equivalent to parent's element font-size
  • 1rem is equivalent to the root element font-size
  • there are way more units but these are the most popular

Bootstrap

The most popular front-end framework

  • HTML/CSS/JavaScript framework
  • Mobile first responsive layouts and grids
  • Tools for typography, forms, buttons, navigation and other interface components
  • Developed at Twitter but open sourced since 2011

Using Bootstrap

Bootstrap can be downloaded from their website getbootstrap.com,
or alternatively used via a CDN: 



<script src="https://code.jquery.com/jquery-latest.js"></script>


<link rel="stylesheet"
    href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.7/css/bootstrap.min.css">


<script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.7/js/bootstrap.min.js">
</script>
To support small screens, the responsive design html5 meta tag should be used 


<meta name="viewport" content="width=device-width, initial-scale=1.0">
A content delivery network(CDN) is a geo distributed network of proxy servers and their data centers.
The goal is to distribute service (page assets) spatially relative to end-users to provide high availability and high performance.

Grid System

Is a responsive mobile-first grid system that scales up to 12 columns as the viewport size increases

Content can span a different number of columns

Grid Classes

Screen sizes are divided into 4 types with different css prefixes

  • col-xs-* - extra small devices (phones, less than 768px, bootstrap default)
  • col-sm-* - small devices (tablets, 768px and up)
  • col-md-* - medium devices (desktops, 992px and up)
  • col-lg-* - large devices (large desktops, 1200px and up)
 


<div class="container-fluid">
    
    <div class="row">
        <div class="col-sm-6 col-lg-4">
            
        </div>
        <div class="col-sm-6 col-lg-4">
            
        </div>
        <div class="col-sm-6 col-lg-4">
            
        </div>
    </div>
</div>
one column on phones (default), two columns on tablets/desktops (size cascades up) and 3 on large desktops

Bootstrap visibility

  • Bootstrap allows you to set the visibility of an element depending on screen size
  • An element can either be shown or hidden with the help of visible and hidden class prefixes

<div class="container-fluid">
    <div class="row">
        <div class="hidden-xs">hide this on phones</div>
        <div class="visible-lg">show this on large desktops</div>
    </div>
</div>
